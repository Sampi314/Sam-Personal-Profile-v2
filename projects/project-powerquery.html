Power Query ETL Pipeline

Step-by-step data transformation demonstration

Overview

Power Query M Code ETL

This demo walks through a complete ETL (Extract, Transform, Load) pipeline built in Power Query. Starting from raw CSV data with missing values and inconsistent formatting, each step progressively cleans, reshapes, and enriches the data until it's ready for analysis. The M code is shown alongside the data preview at every stage.

Pipeline Steps

  • 1. Source: Load raw CSV data and inspect for quality issues (nulls, wrong types).
  • 2. Remove Nulls: Filter out incomplete records using Table.SelectRows.
  • 3. Split Column: Break "Name" into "First Name" and "Last Name" with Splitter.SplitTextByDelimiter.
  • 4. Change Type: Convert text to proper types (numbers, dates) and replace nulls with defaults.
  • 5. Add Column: Create a calculated "Revenue Band" column using conditional M code logic.

Interactive Demo

Click each step in the pipeline to see the data and M code at that stage. Changed cells are highlighted in yellow, removed values in red.

When to Use This

Power Query is the go-to tool for repeatable data transformations in Excel and Power BI. Unlike manual data cleaning, every step is recorded as M code and re-runs automatically when the source data refreshes. I use Power Query extensively at SumProduct to standardize data from multiple sources, clean client-provided datasets, and build robust data pipelines that feed into financial models and reports.
